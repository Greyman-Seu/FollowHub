# FollowHub 产品总览

## 1. 产品定位

FollowHub 是一个面向个人研究者与技术从业者的信息汇聚系统。它的目标不是简单抓取信息，而是每天从多个来源中提取出高价值内容，生成结构化摘要，并以网页形式归档展示。

第一阶段，FollowHub 以本地运行和静态网页输出为核心，优先解决“每天稳定产出一份可读摘要”的问题。后续阶段再逐步扩展到更强的防遗漏能力、公开展示能力，以及接入 Claude、Codex 等 agent 体系。

## 2. 问题背景

当前关注的信息散落在多个平台：

- X 上的大量博主与研究者动态
- 微信公众号文章
- arXiv 每日论文

这些来源具有以下问题：

- 信息量大，噪声多，人工浏览成本高
- 多平台分散，无法统一跟踪
- 需要快速判断“今天先看什么”
- 同时又担心错过重要信息
- 论文主要是英文内容，阅读和筛选成本更高

## 3. 产品目标

FollowHub 的长期目标是成为一个以 agent 为中心、可扩展为 skill 的信息助手系统。

第一阶段目标：

- 每天稳定采集多源信息
- 自动生成一份高信息密度的日报网页
- 对重点内容做中文摘要和价值说明
- 对全量内容保留线索与原文链接
- 支持本地电脑关机后的补跑和恢复

## 4. 目标用户

当前目标用户是项目所有者本人，具备以下特征：

- 长期关注 AI、机器人、论文、开源项目与行业动态
- 每天有较强的信息摄入需求
- 对高价值筛选比“全量抓取”更敏感
- 接受本地运行、静态页面和配置驱动的工作方式

后续可扩展用户：

- 对研究追踪有类似需求的技术读者
- 希望在个人网站公开展示精选信息流的人

## 5. 核心原则

- 重点摘要优先，防遗漏兜底
- 本地优先，不依赖持续在线服务器
- 静态输出优先，方便归档与发布
- LLM 作为增强层，不作为底层基础设施
- 先做稳定 pipeline，再抽象为 skill

## 6. 一阶段产品形态

第一阶段的主要产物是一组静态网页：

- 每日 FollowHub 日报页
- 时间归档页
- 最近日报索引页

这些页面先在本地生成和验证，后续接入 `https://tenstep.top/follow/`。

## 7. 长期演进方向

- 增加更多信息源，如 RSS、GitHub Releases、Newsletter
- 引入更强的兴趣建模和主题聚类
- 将稳定能力抽象为 skill
- 增加主编排 agent
- 将高质量信息进一步沉淀到知识库或 LLM Wiki
