# PRD: Agent / Skill 演进

## 1. 目标

在不影响第一阶段稳定交付的前提下，为 FollowHub 后续接入 Claude、Codex 等 agent 体系预留清晰边界。

## 2. 当前阶段原则

- 第一阶段不以 agent 为核心运行前提
- 第一阶段不要求每层都 skill 化
- 第一阶段优先模块化 pipeline

## 3. 后续可 skill 化的能力

- 单一来源采集
- 单日摘要生成
- 网页生成
- 补跑任务
- 配置检查与更新

## 4. 未来主编排 Agent 的职责

- 根据配置选择要调用的能力
- 根据任务类型选择模型提供方
- 生成日报、周报、专题汇总
- 与知识库或 LLM Wiki 集成

## 5. 模型可插拔要求

- 不同任务支持绑定不同模型
- 模型提供方接口统一
- Claude、Codex、OpenClaw、Hermes 等都应属于可替换的后端

## 6. 成熟条件

只有在以下能力稳定后再 skill 化：

- 输入输出格式稳定
- 错误恢复路径清晰
- 数据依赖边界不再频繁变化
