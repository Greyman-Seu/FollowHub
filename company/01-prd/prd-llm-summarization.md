# PRD: LLM 总结与价值筛选

## 1. 目标

在保留全量线索的前提下，用规则与 LLM 协同完成摘要、相关性判断、价值排序与论文解释。

## 2. 输入

- 采集层输出的标准化条目
- 用户兴趣配置
- 来源优先级配置

## 3. 输出

- 重点内容列表
- 全量线索列表
- 每条内容的简短摘要
- 每条内容的相关性与重要性标记
- 论文的人话解释

## 4. 能力需求

### 4.1 摘要

- 把长内容压缩成简明中文摘要
- 摘要应突出结论和价值，而非机械复述

### 4.2 价值筛选

- 判断一条内容是否值得进入“今日重点”
- 标记重要性和相关性等级

### 4.3 防遗漏兜底

- 未进入重点区的内容仍保留为全量线索
- 除重复、损坏、空内容外，不轻易丢弃

### 4.4 论文专项

每篇论文至少输出：

- 解决什么问题
- 用了什么方法
- 为什么值得关注
- arXiv 原文链接
- `hjfy.top` 翻译链接

## 5. 约束

- LLM 是增强层，不负责底层抓取、状态管理、页面生成
- 模型提供方需要可替换
- 不允许单一大 prompt 承担整个系统逻辑

## 6. 质量要求

- 摘要简洁，不空泛
- 对价值判断给出简短理由
- 对论文解释使用人话，不写学术空话
