# FollowHub 总 PRD

## 1. 产品定义

FollowHub 是一个多源信息汇聚、总结、发布系统。它接收来自 X、微信公众号、arXiv 等平台的原始内容，通过规则与 LLM 协同处理，输出每日重点摘要与全量线索，并生成可长期归档的网页结果。

## 2. 目标与非目标

### 2.1 产品目标

- 降低跨平台信息跟踪成本
- 提升每日关键信息的到达率
- 保留完整线索，避免摘要黑箱导致遗漏
- 形成可公开展示的时间序列网页资产
- 为后续 skill / agent 化预留模块边界

### 2.2 非目标

- 不在第一阶段构建完整知识库
- 不在第一阶段构建聊天式前端
- 不在第一阶段构建多用户或权限系统
- 不在第一阶段依赖长期在线服务器

## 3. 主要使用场景

- 每天定时生成 FollowHub 日报
- 电脑关机若干天后启动并补跑缺失日报
- 在网页中快速浏览当天重点内容
- 回看过去某天的摘要与全量线索
- 为后续网站发布和 agent 接入提供结构化结果

## 4. 核心需求

- 支持多源配置
- 支持定时运行与手动运行
- 支持断点恢复与补跑
- 支持重点筛选与全量保留
- 支持论文专项解释与翻译链接
- 支持生成静态网页与归档索引
- 支持本地日志、错误追踪与重跑

## 5. 核心产出

每天至少产出：

- 一份日报结构化数据
- 一份日报静态页面
- 一份更新后的索引页
- 一份更新后的归档页
- 一份运行状态记录

## 6. 阶段规划

### Phase 1

目标：每天稳定生成一份可读的日报网页。

### Phase 2

目标：增强兴趣建模、减少漏报、优化展示质量。

### Phase 3

目标：将稳定能力 skill 化，由 Claude / Codex 等 agent 编排调用。

### Phase 4

目标：与知识库和更长期的内容组织体系打通。

## 7. 成功指标

第一阶段建议采用以下指标：

- 日报成功生成率
- 补跑成功率
- 重点摘要覆盖率
- 全量线索完整率
- 每日阅读可用性

## 8. 风险

- 来源接入方式不稳定
- LLM 结果质量波动
- 本地运行环境不持续在线
- 网页结构若设计不当，会出现“生成了但不好用”

## 9. 关键产品判断

- 第一阶段优先静态网页，不优先在线服务
- 第一阶段优先本地补跑稳定性，不优先实时更新
- 第一阶段优先模块化 pipeline，不优先全量 skill 化
