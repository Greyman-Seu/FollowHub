# PRD: 第一阶段每日摘要网页

## 1. 目标

第一阶段要解决的问题是：在本地不稳定在线的环境下，每天稳定生成一份可读、可归档、可后续发布的 FollowHub 日报网页。

## 2. 范围

### 2.1 必须做

- 支持 X、公众号、arXiv 三类源
- 支持配置化管理关注源
- 支持每日执行
- 支持启动后补跑最近 7 天内缺失日报
- 支持重点摘要与全量线索双通道输出
- 支持论文中文解释与翻译链接
- 支持生成日报页、索引页、归档页

### 2.2 可简化做

- 来源接入优先复用 RSS 或现有程序
- 价值筛选先采用规则 + LLM
- 网页先保证可读和清晰，不强求复杂交互

### 2.3 不做

- 实时更新
- 多用户系统
- 在线后台管理系统
- 知识库沉淀体系

## 3. 用户体验要求

- 打开当天日报后，首屏能看到今日最值得看的内容
- 重点内容应比全量列表更易扫读
- 任意条目都能快速跳回原文
- 论文条目要能快速理解“解决了什么问题、怎么做的”

## 4. 验收标准

- 有效配置后，可成功生成某天日报
- 断电或停机后可补齐缺失日报
- 网页可长期归档，不依赖数据库才能查看
- 日报同时包含重点区与全量线索区
- 论文条目包含原文链接与翻译链接

## 5. 输出物

- 日报 HTML
- 日报中间结构化数据
- 更新后的索引页与归档页
- 任务日志与状态数据
