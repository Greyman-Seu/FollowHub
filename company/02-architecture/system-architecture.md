# 系统架构

## 1. 总体分层

FollowHub 系统分为五层：

- 采集层
- 理解层
- 发布层
- 运行层
- Agent / Skill 演进层

## 2. 各层职责

### 2.1 采集层

负责从 X、公众号、arXiv 获取原始信息，并统一标准化输出。

### 2.2 理解层

负责去重、相关性判断、重点筛选、摘要生成与论文解释。

### 2.3 发布层

负责将日报结构化结果转为静态网页、索引页和归档页。

### 2.4 运行层

负责调度、补跑、状态记录、日志与失败恢复。

### 2.5 Agent / Skill 层

负责未来以 skill 和 agent 形式调用稳定能力。

## 3. 架构原则

- 底层能力先模块化，再 skill 化
- 运行与状态不依赖单一模型
- 网页输出不依赖在线数据库
- 每层之间使用清晰数据契约通信
